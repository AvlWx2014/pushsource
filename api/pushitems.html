API Reference: push items¶

Common¶

class pushsource.PushItem(*, name: str, state: str = 'PENDING', src: str = None, dest=NOTHING, md5sum: str = None, sha256sum: str = None, origin: str = None, build: str = None, signing_key: str = None)[source]¶

A PushItem represents a single piece of content to be published. This may be an RPM, an advisory, a generic file, and so on.

name = None¶

A name for this push item.

A push item’s name may be any short identifying string which is meaningful to end-users.

In the common case of a regular file, a push item name will simply be a filename, optionally including leading path components.

Container image manifests may provide their digest as the name.

In all cases, a non-empty name must be provided.

state = None¶

The state of this push item.

In the majority of cases, push sources will produce push items with a state of PENDING, with the caller expected to later evolve the state.

In rare cases, push sources may produce push items with other states, such as INVALIDFILE to indicate that the source located invalid content.

src = None¶

The source of this push item.

If the push item is a file, this will be the full path to the file. If the push item does not represent a file, this will generally be omitted.

dest = None¶

Destination of this push item.

The meaning of “dest” differs depending on the source used and its configuration.

Examples for possible uses of “dest” include: - a path to a directory (for items pushed using rsync) - a Pulp repository name (for items pushed using Pulp)

md5sum = None¶

Hex digest of MD5 checksum for this push item, if available.

sha256sum = None¶

Hex digest of SHA256 checksum for this push item, if available.

origin = None¶

A string representing the origin of this push item.

The “origin” field is expected to record some info on how this push item was discovered. The exact semantics depend on the specific source backend in use.

build = None¶

NVR for the koji build from which this push item was extracted, if any.

signing_key = None¶

If this push item was GPG signed, this should be an identifier for the signing key used.

Generally a short key ID such as “F21541EB” is used, though the library doesn’t enforce this.

Errata¶

class pushsource.ErratumPushItem(*, name: str, state: str = 'PENDING', src: str = None, dest=NOTHING, md5sum: str = None, sha256sum: str = None, origin: str = None, build: str = None, signing_key: str = None, type: str = 'bugfix', release: str = '0', status: str = 'final', pushcount: str = '1', reboot_suggested: bool = False, references=NOTHING, pkglist=NOTHING, from_: str = None, rights: str = None, title: str = None, description: str = None, version: str = '1', updated: str = None, issued: str = None, severity: str = None, summary: str = None, solution: str = None, content_types=NOTHING)[source]¶

A push item representing a single erratum (also known as “advisory”).

Note that many fields on erratum objects which appear to be numeric are instead represented as strings (‘release’ and ‘pushcount’ being two examples).

type = None¶

‘bugfix’, ‘security’ or ‘enhancement’.

Type

Type of advisory

release = None¶

Release number. Typically an integer-string, initially ‘0’.

status = None¶

Status, typically ‘final’.

pushcount = None¶

Number of times advisory has been revised and published (starting at ‘1’).

reboot_suggested = None¶

True if rebooting host machine is recommended after installing this advisory.

references = None¶

A list of references (ErratumReference) associated with the advisory.

pkglist = None¶

A list of package collections (ErratumPackageCollection) associated with the advisory.

from_ = None¶

Contact email address for the owner of the advisory.

rights = None¶

Copyright message.

title = None¶

Title of the advisory (e.g. ‘bash bugfix and enhancement’).

description = None¶

Full human-readable description of the advisory, usually multiple lines.

version = None¶

Advisory version. Starts counting at “1”, and some systems require updating the version whenever an advisory is modified.

updated = None¶

Timestamp of the last update to this advisory.

Typically of the form ‘2019-12-31 06:54:41 UTC’, but this is not enforced.

issued = None¶

Timestamp of the initial release of this advisory.

Uses the same format as updated().

severity = None¶

Severity of the advisory, e.g. “low”, “moderate”, “important” or “critical”.

summary = None¶

Typically a single sentence briefly describing the advisory.

solution = None¶

Text explaining how to apply the advisory.

content_types = None¶

A list of content types associated with the advisory.

For example, “docker” may be found in this list if the advisory deals with container images.

class pushsource.ErratumReference(*, href: str, id: str, title: str, type: str = 'other')[source]¶

A reference within a references() list.

href = None¶

A URL.

id = None¶

A short ID for the reference, unique within this erratum.

title = None¶

A title for the reference; analogous to the ‘title’ attribute in HTML.

type = None¶

Type of reference. This defines the expected target of the URL and includes at least:

  • “self”: reference to a page for this advisory

  • “bugzilla”: reference to a bug

  • “other”: any other kind of reference

class pushsource.ErratumPackageCollection(*, name: str, packages=NOTHING, short: str = '', module: pushsource._impl.model.erratum.ErratumModule = None)[source]¶

A collection of packages found within an pkglist().

A non-modular advisory typically contains only a single collection, while modular advisories typically contain one collection per module.

name = None¶

A name for this collection. The collection name has no specific meaning, but must be unique within an advisory.

packages = None¶

List of packages (ErratumPackage) within this collection.

short = None¶

An alternative name for this collection. In practice, this field is typically blank.

module = None¶

An ErratumModule defining the module this entry is associated with, if any.
